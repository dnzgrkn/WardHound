# ADR 0001: Record Architecture Decisions

## Status
Accepted

## Context
We need to record the architectural decisions made on this project so future contributors (and interviewers) understand not just what was built, but why.

## Decision
We will use Architecture Decision Records (ADRs), one per significant decision, numbered sequentially in `docs/adr/`. Each ADR follows: Status, Context, Decision, Consequences.

## Consequences
Every decision with real trade-offs (stack choice, correlation strategy, AI output contract, response engine safety model, etc.) gets its own ADR before or alongside implementation, not retroactively.
